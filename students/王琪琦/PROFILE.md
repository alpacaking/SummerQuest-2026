## 学习与研究

### 最近关注的问题

- 如何将预训练的扩散视频模型改造成自回归长视频生成模型，并缓解长时序生成中的误差累积、画面退化与语义漂移。
- 视频 VAE 的潜空间码率、压缩比例与重建质量之间应如何权衡，以保留更多空间细节和运动信息，并提升后续生成质量。
- 长视频模型训练与推理中的显存、KV Cache 和计算开销如何优化，以及定制 GPU Kernel 和训练基础设施能够带来多大效率收益。


### 过去探索过的方向

- 探索长上下文语言模型的 KV Cache 压缩与高效推理，尝试利用时序信号的频域表示压缩缓存，并结合定制算子优化实际推理效率。
- 研究扩散语言模型的生成与推理方法，包括基于频率信息调整非自回归生成顺序，以及结合高效注意力和在线推理框架优化训练流程。
- 参与多模态模型的推理框架适配，接触了模型部署、算子兼容和推理系统相关的工程问题。
- 弱监督场景文本检测研究，尝试使用框监督、点监督和知识蒸馏改进文本检测方法。
- 调研并整理长上下文语言模型相关工作，重点关注注意力机制、KV Cache 优化与长序列建模。

### 项目与学习经历

#### FourierAttention：面向长上下文模型的 KV Cache 压缩

- 时间：2025.04 - 2026.05
- 背景与问题：长上下文推理中的 KV Cache 会带来较高的显存和计算开销。
- 个人工作：参与方法设计、实验验证与高效算子实现，重点研究不同特征维度的压缩价值。
- 方法与结果：将 KV Cache 的时间演化投影到固定傅里叶基上，并实现 FlashFourierAttention 算子，在维持上下文能力的同时降低缓存开销。
- 局限与反思：当前实验主要覆盖有限的模型与任务，跨架构泛化和算子可移植性仍需进一步验证。

#### FourierSampler：扩散语言模型的频率引导生成

- 时间：2025.10 - 2026.02
- 背景与问题：扩散语言模型的非自回归解码顺序可能影响生成质量。
- 个人工作：参与频率引导生成策略的设计、实验与分析。
- 方法与结果：优先生成低频结构信息，再逐步补充高频细节，在不增加推理时间的情况下提升代码和数学任务表现。
- 局限与反思：目前结论主要来自特定模型和任务，对开放域生成及更大模型的适用性仍需验证。

#### DiRL：扩散语言模型高效后训练

- 时间：2025.11 - 2026.01
- 背景与问题：扩散语言模型后训练存在计算成本高、在线采样效率低的问题。
- 个人工作：参与训练流程优化与实验验证。
- 方法与结果：结合 FlexAttention、LMDeploy 在线更新等方式提升训练效率，并参与适配扩散语言模型的无偏 GRPO 方法 DiPO。
- 局限与反思：系统优化与算法改进同时存在，后续需要更细致地区分各组件带来的独立收益。

#### 弱监督场景文本检测

- 时间：2024.02 - 2025.02
- 背景与问题：降低场景文本检测对精细标注数据的依赖。
- 个人工作：负责框监督、点监督方法调研，并开展模型改进和知识蒸馏实验。
- 方法与结果：完成对 BoxSnake 方法的改进，项目获得优秀结项。
- 局限与反思：实验主要面向特定数据与监督形式，跨场景鲁棒性和实际部署效果仍需进一步评估。

## CS336 学习计划

- 当前基础：使用过 Python、PyTorch 和 Hugging Face 生态开展模型训练与推理实验，接触过长上下文模型、扩散语言模型、多模态推理适配和 Triton 算子开发。目前对模型推理优化相对熟悉，但尚未系统完成从 tokenizer、预训练数据处理到分布式训练和 scaling law 的完整流程，希望补齐大模型预训练与训练基础设施方面的知识。
- A1 Basics：建立可复现的基础训练流程。
- A2 Systems：使用 profiler 分析训练过程中的计算、显存、数据加载与通信瓶颈，理解混合精度、算子融合和 GPU Kernel 优化；尝试比较 PyTorch 实现与 Triton 算子的性能，为长视频生成中的高效注意力和训练优化积累经验。
- A3 Scaling：学习 scaling law 的实验设计、拟合和资源分配方法，关注拟合残差、外推误差以及不同模型规模、数据规模和计算预算之间的关系。
- A4 Data：实践语料清洗、去重、质量过滤和数据配比，比较处理前后的长度、领域和质量分布；重点理解数据组成对预训练结果的影响，并探索相关经验如何迁移到视频数据的采样、过滤和时序组织中。
- A5 Alignment：学习 SFT、偏好优化和 RLVR 等基本方法，关注奖励设计、训练稳定性与 reward hacking；通过失败样例分析模型究竟学到了目标能力，还是利用了评测或奖励中的漏洞。
- A6 Harness：计划重点关注实验配置管理、自动化评测、日志记录和结果复现，使不同模型、数据和系统配置能够进行可靠比较。
